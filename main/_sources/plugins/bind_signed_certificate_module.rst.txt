.. Title

cisco.ise.bind_signed_certificate module -- Resource module for Bind Signed Certificate
+++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `cisco.ise collection <https://galaxy.ansible.com/cisco/ise>`_ (version 3.0.1).

    To install it, use: :code:`ansible-galaxy collection install cisco.ise`.
    You need further requirements to be able to use this module,
    see :ref:`Requirements <ansible_collections.cisco.ise.bind_signed_certificate_module_requirements>` for details.

    To use it in a playbook, specify: :code:`cisco.ise.bind_signed_certificate`.

.. version_added

.. rst-class:: ansible-version-added

New in cisco.ise 1.0.0

Synopsis
--------

.. Description

- Manage operation create of the resource Bind Signed Certificate.
- Bind CA Signed Certificate.

.. note::
    This module has a corresponding :ref:`action plugin <action_plugins>`.

Requirements
------------
The below requirements are needed on the host that executes this module.

- ciscoisesdk \>= 2.0.1
- python \>= 3.5

.. Notes

Notes
-----

.. note::
   - SDK Method used are certificates.Certificates.bind\_csr,
   - Paths used are post /api/v1/certs/signed-certificate/bind,
   - Does not support \ :literal:`check\_mode`\ 
   - The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager from Cisco ISE SDK
   - The parameters starting with ise\_ are used by the Cisco ISE Python SDK to establish the connection
